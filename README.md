# 2T509 データ構造論 グループ3 システム説明書
今回は提出されたシフト希望を読み込んで自動的にシフトを均等に割り振り表示する機能を実装した．ページ上では他のボタンも作ってはいるがいずれも未実装である
## 起動方法
- week7の下のMARUNAGEディレクトリに移動して"gradlew bootRun"または".\gradlew bootRun"を実行
- "http://localhost:2289"のURLに飛ぶと見れるようになる

## 操作説明
### 従業員側
- "シフト希望提出"ボタンから自分のシフト希望を提出できる(未実装)
- "シフト希望一覧を確認"ボタンから従業員が出しているシフト希望を表で確認できる
- "確定シフト確認"ボタンから最終的なシフトを確認できる(未実装)
### 雇用主側
- "従業員登録"で従業員の新規登録，やめた従業員の削除ができる(未実装)
- "シフト自動生成・確認"ボタンで自動的に割り振ったシフトを確認できる．今回は21人分のシフト希望があらかじめ書かれたcsvファイルを用意し，それを読み込むようにした．これを見て雇用主は微調整をし，シフトを確定させる(微調整および確定機能は未実装)

## 自動割り振りのアルゴリズム
### シフト希望があるコマの場合
- まず特定の1コマ(7/21(月)朝のキッチンなど)を希望している従業員を探す
- その従業員が目標の3コマ(63÷21=3)に達していないか確認する
- その従業員が同じ時間帯で別のポジションに割り振られていないか確認する
- この条件を満たす人が複数人いた場合はシフト数が最も少ない人を優先する
### シフト希望がないコマの場合
シフト数が少ない人を優先的に割り振って埋める．このときもポジションが掛け持ちにならないようになっている
